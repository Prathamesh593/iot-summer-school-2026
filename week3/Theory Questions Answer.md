# Q29. Sensor Calibration in IoT & the Two-Point Method

## What is Sensor Calibration?

Sensor calibration is the process of comparing a sensor's output with a known reference value and adjusting the readings to improve accuracy.

## Why is Sensor Calibration Important?

- Ensures accurate and reliable sensor readings.
- Compensates for manufacturing variations between sensors.
- Reduces errors caused by temperature, humidity, and sensor aging.
- Improves the quality of data used in IoT applications.
- Prevents incorrect decisions in automation systems.

---

## Calibrating an Analog Soil Moisture Sensor

An analog soil moisture sensor provides raw ADC values (typically **0–1023** on Arduino). These values must be converted into a meaningful **0%–100% moisture level** using calibration.

---

## Two-Point Calibration Method

The Two-Point Calibration Method assumes the sensor response is linear and uses two known reference points.

### Step 1: Dry Calibration (0% Moisture)

- Keep the sensor completely dry in the air.
- Record the raw ADC value.

**Example:**

```text
Raw_dry = 850
```

This represents:

```text
0% Moisture
```

---

### Step 2: Wet Calibration (100% Moisture)

- Immerse the sensor in water (up to the recommended level).
- Record the raw ADC value.

**Example:**

```text
Raw_wet = 290
```

This represents:

```text
100% Moisture
```

---

### Step 3: Convert Raw Value to Moisture Percentage

For any sensor reading:

```text
Raw_current
```

Calculate moisture percentage using:

```text
Moisture (%) =
((Raw_current - Raw_dry) /
(Raw_wet - Raw_dry)) × 100
```

**Example**

```text
Raw_dry     = 850
Raw_wet     = 290
Raw_current = 500

Moisture (%) =
((500 - 850) / (290 - 850)) × 100
= 62.5%
```

> **Note:** Capacitive soil moisture sensors usually produce **higher values when dry** and **lower values when wet**. The formula automatically accounts for this.

# Q30. I2C Protocol in IoT Sensor Communication

## What is I2C?

**I2C (Inter-Integrated Circuit)** is a serial communication protocol used to connect multiple sensors and devices to a microcontroller using only **two wires**.

It is commonly used with Arduino, ESP32, Raspberry Pi, and other IoT development boards.

---

## SDA and SCL Lines

I2C communication uses two signal lines:

### SDA (Serial Data Line)

- Transfers data between the master and slave devices.
- Bidirectional communication line.

### SCL (Serial Clock Line)

- Provides the clock signal for synchronization.
- Generated by the master device.

---

## How I2C Addressing Works

Since multiple devices share the same two wires, each device must have a unique address.

### Communication Process

1. The master sends a **Start Condition**.
2. The master sends the **7-bit slave address**.
3. The master sends a **Read (1)** or **Write (0)** bit.
4. The addressed slave responds with an **ACK (Acknowledgment)**.
5. Data is transmitted between the master and slave.
6. Communication ends with a **Stop Condition**.

---

## Advantages of I2C

- Requires only two communication wires.
- Supports multiple devices on the same bus.
- Simple wiring and low hardware cost.
- Widely supported by IoT sensors and microcontrollers.

---

## Common IoT Sensors Using I2C

| Sensor | Function |
|---------|----------|
| **BME280 / BMP280** | Temperature, Humidity, and Pressure |
| **MPU6050** | Accelerometer and Gyroscope |
| **BH1750** | Digital Light Intensity (Lux) Sensor |

---

## Summary

- **Sensor Calibration** improves measurement accuracy and reliability in IoT systems.
- The **Two-Point Calibration Method** converts raw sensor values into meaningful percentages using dry and wet reference points.
- **I2C** is a simple two-wire communication protocol widely used for connecting multiple sensors.
- **SDA** carries data, while **SCL** provides the clock signal.
- Each I2C device has a unique address, allowing multiple sensors to communicate on the same bus.
