# Project 6: Wi-Fi Weather Dashboard

## Components Used
- ESP32 Dev Module (30-pin)
- AM2302 (DHT22) Temperature & Humidity Sensor
- BMP280 Pressure/Temperature Sensor
- LM393 Photosensitive LDR Sensor Module (4-pin, analog output)
- 0.96" SSD1306 OLED, 128x64, 7-pin SPI/I2C combo (wired in SPI mode)

## Wiring Diagram (text)

**DHT22:**
- VCC → 3.3V, GND → GND, DATA → GPIO4

**BMP280:**
- VCC → 3.3V, GND → GND, SDA → GPIO21, SCL → GPIO22 (I2C address 0x76)

**LDR Module:**
- VCC → 3.3V, GND → GND, AO → GPIO34

**OLED (SPI mode):**
- VCC → 3.3V, GND → GND
- D0 (CLK) → GPIO18, D1 (MOSI) → GPIO23
- RES → GPIO17, DC → GPIO16, CS → GPIO5

## Libraries Used
- DHTesp by beegee_tokyo
- Adafruit BMP280
- Adafruit SSD1306, Adafruit GFX Library, Adafruit BusIO
- AsyncTCP (ESP32Async fork)
- ESPAsyncWebServer (ESP32Async fork)

## Before You Upload — Wi-Fi Credentials Setup
This sketch requires a `config.h` file in the same folder as the `.ino`, containing your own Wi-Fi credentials (not included in this repository for security reasons).

Create a file named `config.h` in this same folder with the following content:
```cpp
#define WIFI_SSID "your_actual_wifi_name"
#define WIFI_PASSWORD "your_actual_wifi_password"
```

**Important:** ESP32 only supports **2.4GHz** Wi-Fi networks — 5GHz-only networks will fail to connect.

## How to Run
1. Create `config.h` as described above, in the same folder as the `.ino` file
2. Open `p6-wifi-dashboard.ino` in Arduino IDE
3. Board: ESP32 Dev Module
4. Select correct COM port
5. Upload
6. Open Serial Monitor at 115200 baud — note the printed IP address (format: `192.168.X.X`)
7. On any device connected to the **same Wi-Fi network**, open a browser and navigate to `http://<printed-ip>/`

## Expected Output
- OLED shows "Connecting WiFi...", then SSID/IP for a few seconds, then cycles live sensor readings every 5 seconds
- Web dashboard displays 4 live sensor cards (Temperature, Humidity, Pressure, Light), color-coded by temperature range, auto-refreshing every 30 seconds
- `/data` endpoint returns live JSON: `{"temp":28.5,"humidity":65,"pressure":1013.2,"light":72}`
- `/history` endpoint returns the last 24 temperature readings (5-minute intervals) as JSON, used to render a live line chart on the dashboard (Chart.js, loaded via CDN)
- Multiple devices on the same network can load the dashboard simultaneously and see matching live data

## Local IP Format
The ESP32 prints its assigned IP address to Serial Monitor at boot, in the format `192.168.X.X` (typical home router range), depending on your router's DHCP configuration. This address is only reachable by devices on the same local Wi-Fi network — it is not accessible from outside your home network or over mobile data.

## Bonus Challenge — Implemented
24-point temperature history chart (5-minute intervals) rendered as a live line graph using Chart.js (loaded via CDN), served from a dedicated `/history` JSON endpoint on the ESP32.

## Build Notes
- ESP32 only supports 2.4GHz Wi-Fi networks; 5GHz networks are not compatible and will fail to connect.
- BMP280 I2C address confirmed as `0x76` via I2C scanner sketch before use.
- LDR module used is an LM393-based comparator breakout with onboard voltage divider (analog output used directly, no external resistor needed).
- Chart.js requires the viewing device (phone/laptop browser) to have internet access, since it is loaded from a CDN; the ESP32 itself only requires local Wi-Fi, not internet access.
- Wi-Fi credentials are kept out of this repository via `config.h`, which is intentionally not included — see setup instructions above.
